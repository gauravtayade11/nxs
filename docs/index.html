Built for real production workflows
Three capabilities that go beyond basic debugging.
🔮
Failure Prediction
nxs predict -n production
Scans your entire cluster for pods that are about to fail — before PagerDuty fires. Two detection passes: pod state (always works) + resource usage (kubectl top). No metrics-server required.
  • ✗ CRITICALOOMKilled — memory at 94% of limit, kill imminent
  • ✗ CRITICALCrashLoopBackOff — 142+ restarts
  • ⚠ WARNINGRequest/limit gap 32Mi → 256Mi (8× mismatch)
  • ⚠ WARNINGImagePullBackOff — bad tag or missing credentials
  • ⚠ WARNINGPending — can't schedule (node pressure / taints)
  • → FIXExact kubectl command per pod — real names, no placeholders
⚙️
CI/CD Pipeline Debugger
nxs devops pipeline --stdin
Paste a failing GitHub Actions, Jenkins, or CircleCI log and get root cause + fix steps in seconds. Understands build failures, test failures, deploy errors, and timeout patterns.
  • ✗npm ERR! — dependency conflict, missing lock file
  • ✗Out of memory — runner killed Java/Node process
  • ✗Permission denied — missing IAM role or secret
  • ⚠Timeout exceeded — flaky test or slow build step
  • ⚠Docker build failed — bad Dockerfile layer or cache miss
  • → FIXExact step to fix, which file to change, which env var to add
🔒
Container Security Scanner
nxs sec scan --image nginx:latest
Wraps Trivy to scan Docker images and running pods for CVEs. AI prioritizes which vulnerabilities actually matter and gives you the exact fix command — not just a raw CVE list.
  • ✗ CRITICALCVE-2024-xxxx — remote code execution in base image
  • ✗ HIGHOutdated OpenSSL — known exploit available
  • ⚠ MEDIUM3 medium CVEs in npm dependencies
  • → FIXUpgrade base image: FROM node:18-alpine → node:20-alpine
  • → FIXnpm audit fix --force for dependency CVEs
  • ✓ SCANWorks on image name, running pod, or entire namespace
